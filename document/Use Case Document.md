## 详细用例描述

### 用例1

|   条目   | 内容                                       |
| :----: | :--------------------------------------- |
|   ID   | 1                                        |
|  创建者   | 訾源                                       |
|   时间   | 2017.9.10                                |
|   名称   | 商品分类管理                                   |
| 最后更新者  | 訾源                                       |
| 最后更新日期 | 2017.10.15                               |
|  参与者   | 库存管理人员。目的是为了简便、规范化的完成商品分类的管理，包括增加分类、删除分类、修改分类。 |
|  触发条件  | 库存管理人员对商品分类进行管理。                         |
|  前置条件  | 当前登录账号为库存管理人员账号，身份已验证无误。                 |
|  后置条件  | 新的商品分类被保存。                               |
|  优先级   | 中                                        |
|  正常流程  | 1 系统显示根分类下的商品及商品分类信息，包括编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价 2 库存管理人员请求进入当前分类下的子分类中 3 系统显示当前分类下的商品及商品分类信息，包括编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价 4 库存管理人员请求返回父级分类 5 系统显示父级分类下的商品及商品分类信息，包括编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价 <br>6.1<br>&emsp;6.1.1 库存管理人员请求添加商品分类，选择添加位置，输入新的商品分类名称<br> 6.1.2 系统保存新的商品分类，显示最新的商品及商品分类信息。<br>6.2<br>&emsp; 6.2.1 库存管理人员请求修改商品分类，选择要修改的分类，输入新的分类名称<br> 6.2.2 系统保存修改后的的商品分类，显示最新的商品及商品分类信息<br>6.3<br>&emsp; 6.3.1 库存管理人员请求删除分类，选中分类，进行删除操作<br>&emsp; 6.3.2 系统删除分类，显示最新的商品及商品分类信息 7 库存管理人员以任意顺序重复上述步骤2，4，6或其中的部分步骤，直到所有的商品分类操作完成 |
|  扩展流程  | 4a 在根分类下请求进入父分类：<br> 系统提示当前分类没有父分类，返回正常流程步骤1 6a. 在已经存在商品的分类下添加子分类：<br>&emsp;系统提示分类不能添加在存在商品的分类下，返回正常流程第6步。<br> 6b. 删除已经存在商品/分类的分类：<br>&emsp;系统提示分类被删除的分类不能含有子分类或商品，返回正常流程第6步。<br> 6c. 添加已经存在的分类：<br>&emsp;  系统提示分类添加时不能添加当前目录下已经存在的分类，返回正常流程第6步。<br>6d. 修改后分类名和当前分类下的其他分类名称相同：<br>&emsp;系统提示修改后分类不能和当前分类下其他分类名称相同，返回正常流程第6步 |
|  特殊需求  | 简单易用，无操作难度。                              |

|  触发条件  | 库存管理人员对商品进行管理。                           |
|  前置条件  | 当前登录账号为库存管理人员账号，身份已验证无误。                 |

|  优先级   | 高                                        |
|  正常流程  | 1 系统显示根分类下商品及商品分类信息，包括各个分类下商品的属性，包括编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价 2 库存管理人员请求进入当前分类下的子分类中 3 系统显示当前分类下的商品及商品分类信息，包括编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价 4 库存管理人员请求返回父级分类 5 系统显示父级分类下的商品及商品分类信息，包括编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价<br>6.1<br>&emsp; 6.1.1 库存管理人员选择添加商品， 输入新商品的名称、型号 <br>&emsp;6.1.2 系统自动生成编号，用0填充库存数量、进价、零售价、最近进价、最近零售价，保存新添加商品的信息，显示添加商品后的商品列表 <br>6.2<br>&emsp;6.2.1 库存管理人员选择修改商品，选中要修改的商品，在要修改的属性处输入新的属性<br>&emsp; 6.2.2 系统保存修改后的信息，显示新的商品信息列表  <br>6.3<br>&emsp;6.3.1 库存管理人员选择删除商品，选中商品后删除 <br>&emsp;6.3.2 系统删除商品信息，显示新的商品信息列表<br>6.4<br>&emsp; 6.4.1 库存管理人员选择查询商品，输入商品的编号或者商品的名称，进行查询 <br>&emsp;6.4.2 如果存在匹配的商品，系统显示匹配的商品条目，否则显示空 7 以任意顺序重复上述步骤2，4，6或其中的部分步骤，直到所有的商品分类操作完成 |
|  扩展流程  | 4.a 在根分类下请求进入父分类：<br> 系统提示当前分类没有父分类，返回正常流程步骤1 6.1a. 添加已有的商品（根据商品名称、型号来进行确定是否相同） 在有分类的目录下：<br>&emsp;  系统提示商品添加必须和其他商品无冲突并且只能在没有分类的目录下添加并提示错误原因（如和哪些商品冲突），返回正常流程第6步。<br> 6.2b. 修改商品信息时出现不合法内容/修改后商品信息和其他商品信息冲突：<br>&emsp;系统提示商品信息修改错误原因（如和哪些商品冲突），返回正常流程第6步。6.3c.删除库存不为0或存在交易记录的商品：<br>&emsp;系统提示删除不能删除存在交易记录或者库存不为0的商品，返回正常流程第6步 |
|  特殊需求  | 简单易用，无操作难度。                              |
